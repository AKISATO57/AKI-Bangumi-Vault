设置与登录

当前版本是单文件网页应用：不用 Python、不用命令行，双击这个 HTML 就能打开。日常“同步最近”只扫描最近自定义月份内更新的收藏；“完整同步”才会核对全部条目并检查疑似消失。

用户名用于读取 /v0/users/{username}/collections。
日常同步会从每个分类/状态读取最近 N 个月更新的条目，减少 API 请求。
主题模式
点击切换时会用扩散动画过渡；跟随系统会随系统明暗变化。
字体风格
使用本机已安装字体；没有对应字体时自动回退。
海报卡片
卡片文字区固定高度，悬停时弹起展开，不再把整行撑乱。
动效强度
页面卡顿时可改成轻量或关闭；不会影响备份数据。
越小越顺滑；越大越少点“再显示”。
设为 0 可基本隐藏预览，只在详情里看完整吐槽。
控制卡片、弹窗、按钮等玻璃边框；设为 0 可关闭大部分白边。
安全建议：便携版启动时会把收藏库保存到 资料库/收藏数据.json，把封面保存到 资料库/封面缓存。token 只会发往 api.bgm.tv；不勾选“记住 token”时不会写入 收藏数据.json。

同步中

日常同步只读取最近几个月的更新并增量写入；完整同步才做全量核对和安全箱判断。

导出与导入

完整 ZIP 包含 JSON、CSV、离线 HTML、Word、XLSX 表格、历史版本、同步记录与 images 封面目录。为避免大量 Base64 导致浏览器卡死，完整 ZIP 使用真实图片文件；请先解压 ZIP，再打开 offline-poster-wall.html。

封面离线取决于是否已缓存。完整 ZIP 会把 资料库/封面缓存 里的封面作为 images/ 文件夹打包，速度更快、不会塞满 HTML；请解压后打开 offline-poster-wall.html。单文件 HTML 会把封面内嵌成 Base64，条目很多时会很慢，只建议少量条目使用。
